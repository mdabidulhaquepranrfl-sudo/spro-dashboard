# Frontend Preservation Rules

## Important Instructions

The frontend UI/UX design is already finalized and approved.

Do NOT change:

* Existing layout structure
* Spacing and alignment
* Colors and gradients
* Typography and font sizing
* Card designs
* Responsive behavior
* Animations and transitions
* Tailwind class structure unless absolutely necessary
* Existing visual appearance in any viewport

## Allowed Changes

You may:

* Refactor code into reusable components
* Improve functionality
* Add business logic
* Optimize state management
* Split large files into smaller components
* Improve maintainability
* Add API integration
* Improve performance

## Strict Restriction

While refactoring or adding functionality, the final rendered UI must remain visually identical to the current design.

No unintended visual changes are allowed.
Any new component must preserve the exact existing design and behavior.
